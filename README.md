# Markdown Viewer

An offline-first Next.js PWA for editing Markdown files and keeping local notes. It supports direct file-system access in Chromium browsers, live preview, local persistence, install prompts, and a custom service worker.

## Quick Start

Install dependencies:

```bash
pnpm install
```

Run the app locally:

```bash
pnpm dev
```

Open `http://localhost:3000`.

Create a production build:

```bash
pnpm build
```

Check lint before committing:

```bash
pnpm lint
```

## Scripts

```bash
pnpm dev
pnpm build
pnpm start
pnpm lint
```

Open `http://localhost:3000` during development.

## Project Structure

```text
apps/web/                         Next.js PWA wrapper, portal shell, route handlers, backend, assets
apps/web/src/app/                 App Router routes, layout, API adapters, metadata
apps/web/src/features/portal/     Portal shell, login, session-level app composition
apps/web/src/features/pwa/        PWA install prompt, offline banner, service-worker update UI
apps/web/backend/                 Supabase client, DB migrations, server handlers
apps/web/public/                  Static assets, icons, and service worker
packages/common/                  Shared utilities and browser typings used by workspace packages
packages/markdown-editor/         Markdown editor package consumed by the web app
packages/note-taking/             Note-taking package consumed by the web app
```

The web app imports Markdown from `@markdown-viewer/markdown-editor` and Notes from `@markdown-viewer/note-taking`. Both feature packages depend on `@markdown-viewer/common` for shared utilities and browser typings. App-local imports use the `@/*` alias, backend imports use `@backend/*`, and files inside the same feature use local relative imports.

## PWA

The web app manifest is generated by `apps/web/src/app/manifest.ts`. The service worker lives at `apps/web/public/sw.js`.

Notes support offline editing in the installed PWA/browser. The notes workspace keeps a local IndexedDB cache and queues creates, edits, and deletes while offline. When the browser comes back online, queued changes are synced to the Supabase-backed API.

## Supabase Notes Backend

For first-time setup, run `apps/web/backend/supabase/init.sql` in your Supabase SQL editor. It creates the `public.notes` table, defaults, indexes, update trigger, and reloads the Supabase schema cache.

If you use migrations, the same schema is available at `apps/web/backend/supabase/migrations/001_create_notes.sql`.

Add these environment variables locally and in your deployment:

```bash
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
```

The service role key is used only by Next.js API routes. Do not expose it with a `NEXT_PUBLIC_` prefix.

## Vercel Backend Deployment

The backend runs as Next.js Route Handlers on Vercel. The files under `apps/web/src/app/api` are thin adapters, and the implementation lives in `apps/web/backend/`.

Before deploying:

1. Run `apps/web/backend/supabase/init.sql` in the Supabase SQL Editor.
2. Add these variables in Vercel Project Settings > Environment Variables:

```bash
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-or-secret-key
```

3. Redeploy after adding or changing environment variables.
